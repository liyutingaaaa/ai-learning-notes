# 架构设计文档

## 系统架构

```
┌─────────────────────────────────────────────────────────┐
│                    Vue 3 前端 (Vite)                    │
│   ┌──────────────────────────────────────────────────┐  │
│   │  页面组件  │  业务逻辑  │  状态管理 (Pinia)       │  │
│   └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │
                  HTTP / REST API
                           │
┌─────────────────────────────────────────────────────────┐
│              Spring Boot 后端 (REST API)                │
│   ┌──────────────────────────────────────────────────┐  │
│   │  Controller  │  Service  │  Repository (MyBatis)│  │
│   └──────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
                           │
                        JDBC
                           │
        ┌──────────────────────────────────┐
        │      MySQL 8.0 数据库             │
        │  ├─ users 用户表                │
        │  ├─ notes 笔记表                │
        │  ├─ categories 分类表           │
        │  ├─ tags 标签表                 │
        │  ├─ ai_records AI记录表          │
        │  └─ study_stats 统计表          │
        └──────────────────────────────────┘
```

## 核心模块设计

### 1. 用户模块
- 用户注册、登录、登出
- JWT 认证
- 用户信息管理

### 2. 笔记模块
- 笔记的 CRUD 操作
- 笔记分类管理
- 笔记搜索和过滤

### 3. 标签模块
- 标签的 CRUD 操作
- 标签-笔记关联管理

### 4. AI 模块
- 笔记总结
- 关键词提取
- 题目生成
- 学习建议

### 5. 统计模块
- 学习时长统计
- 笔记数量统计
- AI 调用统计

## 数据流

### 创建笔记流程
```
1. 用户输入笔记内容 (前端)
2. 调用 POST /api/notes (前端)
3. 后端验证 JWT 和数据合法性
4. 保存到数据库
5. 返回创建成功的笔记对象
6. 前端更新本地状态
```

### AI 总结流程
```
1. 用户点击"AI 总结"按钮
2. 调用 POST /api/ai/summarize?noteId=xxx (前端)
3. 后端获取笔记内容
4. 调用 AI 服务 (OpenAI/本地 LLM)
5. 保存 AI 记录到数据库
6. 返回总结结果
7. 前端显示结果并保存到笔记
```

## 安全设计

1. **JWT 认证**: 所有受保护接口都需要有效的 JWT Token
2. **密码加密**: 使用 BCrypt 加密存储用户密码
3. **CORS 配置**: 严格的跨域资源共享策略
4. **输入验证**: 所有用户输入都进行校验
5. **SQL 注入防护**: 使用 MyBatis 参数化查询

## 性能优化

1. **数据库索引**: 在 user_id, created_at 等字段建立索引
2. **全文搜索**: MySQL 全文索引支持笔记搜索
3. **缓存**: 考虑使用 Redis 缓存热点数据
4. **分页**: 列表接口都支持分页
5. **CDN**: 前端静态资源可部署到 CDN
